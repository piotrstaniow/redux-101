What's an application state?

All the data an application needs at a given moment:

  • To display User Interface of given structure and content
    e.g. user input, preflight data, connected devices, events
  • To perform actions and calculations
    e.g. authentication tokens, cached images and responses
  • To provide behaviour persistency
    e.g. responses from APIs, data from sockets

What Redux actually is?

  • Lightweight, reliable and scalable application architecture (e.g. Facebook)
  • A small set of building blocks used to manage the application state
  • A way of thinking about where to put business logic

Three principles of Redux

These are the intents of Redux

  • Single Source of Truth
  • Readonly State
  • State is updated with Pure Functions (i.e. reducers)
